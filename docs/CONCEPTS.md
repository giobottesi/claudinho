# How It Works: Concepts & File Organization

## Project Structure

```
your-repo/
├── README.md                          # Main entry point
└── .github/
    ├── agents/
    │   ├── copilot-instructions.agent.md          # Main agent definition
    │   └── references/
    │       ├── dev-profile-TEMPLATE.md            # Template for personal profiles
    │       ├── dev-profile-patterns.md            # Generic patterns reference
    │       ├── [your-name]-profile.md             # Example personal profile
    │       └── README.md                          # Guide to profiles
    │
    ├── skills/
    │   ├── setup-agent/SKILL.md                   # Code scanner + questionnaire
    │   ├── data-analysis/SKILL.md                 # SQL, trends, anomalies
    │   ├── explain/SKILL.md                       # Learning & explanations
    │   ├── capture-learning/SKILL.md              # Knowledge management
    │   └── brag/SKILL.md                          # Work documentation
    │
    ├── COPILOT_INSTRUCTIONS_TEMPLATE.md           # Template that setup-agent fills
    └── copilot-instructions.md                    # Generated (auto-filled by setup-agent)
```

---

## File Purposes

### `.github/agents/copilot-instructions.agent.md`
**The main agent definition.** Describes:
- What this agent does
- When to use it
- Auto-delegation rules (when to call which skill)
- Key capabilities

Think of it as the "personality" of your project's AI assistant.

### `.github/agents/references/`
**Reference materials for developer preferences and patterns:**

- `dev-profile-TEMPLATE.md` — Blank template for creating personal profiles
- `dev-profile-patterns.md` — Generic, reusable patterns (response directives, code patterns, communication styles)
- `[your-name]-profile.md` — Personal profiles (e.g., examples, team members' preferences)
- `README.md` — Guide explaining how profiles work

Use these to understand and customize your development preferences.

### `.github/skills/SKILL.md`
**Specialized tools.** Each skill is independent:

| Skill | Purpose |
|-------|---------|
| `setup-agent` | Scan code + ask 5 questions → generate instructions |
| `data-analysis` | SQL, pandas, trends, cohort analysis |
| `explain` | Deep dives into systems, concepts, papers |
| `capture-learning` | Personal knowledge management, patterns |
| `brag` | Document work with evidence for portfolios |

Invoke via `/skill-name` in Copilot Chat.

### `.github/copilot-instructions.md`
**Your auto-generated context file.** Contains:
- Project info (name, purpose, tech stack)
- Code conventions (naming, file organization)
- Known gotchas (what breaks often)
- External integrations (APIs, services)
- Debugging style (how your team debugs)
- Domain jargon (team terminology)

This file is **generated by `/setup-agent`** and should be **committed to git**.

### `.github/COPILOT_INSTRUCTIONS_TEMPLATE.md`
**The template that `/setup-agent` fills.** Contains placeholders for the 5 setup questions and auto-detected patterns. Used by setup-agent to generate your project's `.github/copilot-instructions.md`.

## The Setup Process (3 Phases)

### Phase 1: Code Scanning (Automatic)
`setup-agent` scans your repo and detects:
- **Tech stack** — From package.json, imports, Gemfile, etc.
- **File organization** — Where models, services, tests live
- **Naming patterns** — How classes/functions are named
- **Build commands** — npm, bin/rails, cargo, etc.
- **Main domains** — The systems your project builds

**Result**: ~80% of context established automatically, zero questions asked.

### Phase 2: 5 Personal Questions (Human Input)
Then **you** answer 5 questions that code *cannot* know. (See [QUICK_START](../docs/QUICK_START.md) for examples.)

**Takes**: ~5 minutes to answer sincerely.

### Phase 3: Auto-Generate
`setup-agent` creates `.github/copilot-instructions.md`:
- Combines detected info + your answers
- Formats into clean markdown
- Ready to commit immediately

**Output**: File is copy-paste ready, no editing needed.

## How Your Team Uses It

### New Engineer Onboarding

**Day 1:**
```bash
git clone your-repo
cd your-repo
code .

# In Copilot Chat:
/setup-agent
```

Answer 5 questions (5 min) OR confirm existing answers (2 min).

**Result**: Copilot understands your codebase, conventions, and gotchas immediately.

### Updating When Things Change

When you significantly update your codebase:
```
/setup-agent Update — we switched to TypeScript, added Kubernetes
```

Asks *only* for changed answers. Keeps previous answers you don't change.

## Agent Delegation Pattern

Your main agent **auto-delegates** to specialists based on request type:

```
User request to Copilot Chat
    ↓
Main agent (copilot-instructions.agent.md) reads context
    ↓
Detects request type:
    ├→ "Analyze error frequency" → /data-analysis
    ├→ "Explain how caching works" → /explain
    ├→ "Save this pattern" → /capture-learning
    ├→ "Document this work" → /brag
    └→ General coding question → Handles directly
    ↓
Specialized skill gives expert answer
    ↓
Back to main agent (optional)
```

**You can also invoke directly**:
- `/data-analysis` — Start data analysis flow
- `/explain` — Ask for explanations
- Type `@` to see all available agents/skills

## Naming Conventions

- **Skills**: kebab-case, action-noun (`setup-agent`, `data-analysis`, `capture-learning`)
- **Files**: UPPERCASE.md for documentation, PascalCase for agent names
- **Domains**: Categories in your codebase (authentication, payments, etc.)

## Customization

### Adding Your Own Skill

Create `.github/skills/your-skill/SKILL.md`:
```
---
name: your-skill
description: "What this skill does"
---

# Your Skill

...content...
```

Reference it from your main agent.

### Updating Agent Behavior

Edit `.github/copilot-instructions.agent.md` to add:
- New delegation rules
- Project-specific patterns
- Custom capabilities

---

## Key Insight

**The goal**: Don't make engineers explain "how our codebase works" again for each AI tool.

Instead:
- ✅ Build context **once** (run `/setup-agent`)
- ✅ Commit to git (share with team)
- ✅ Use everywhere (Copilot, future tools)
- ✅ Update as you grow

Result: Consistent, team-wide AI assistant that understands your project.
